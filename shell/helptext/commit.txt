commit - commit a play in the given game, and move to the next turn

Example usage:

    commit #4
    commit 3J FOO
    commit K5 BAR.D
    commit exchange ABC
    commit pass

If you pass in a number, it refers to the number of the play
in the list of plays. You can see the current list of plays with the list
command, and you can generate plays to add to the list with the gen
command.

You can also commit a play that you would have otherwise added with the `add`
command, using the same parameters as `add`. See the help for the `add` command.

Flag: -tileorder <letters>

    Specify the exact bag draw order for this commit. Letters are listed
    left-to-right in draw order (first drawn leftmost). The committing player
    draws the first N tiles from the sequence to refill their rack; the rest
    stay in the bag. The opponent's rack is automatically set from the
    remaining unseen tiles.

    The tileorder length must equal (unseen_pool_size - 7), i.e. the expected
    bag size after the opponent is dealt a full rack. Shorter is ambiguous;
    longer is invalid.

    Example: after a PEG solve shows "tiles=HA outcome=win", replay that
    specific eventuality:

        commit 11D ...D -tileorder HA
