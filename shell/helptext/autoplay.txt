autoplay [config-file.json] [overrides] - run a computer vs computer experiment

Usage:
    autoplay my-experiment.json
    autoplay my-experiment.json -numgames 500 -tau1 0.008
    autoplay -botcode1 HASTY_BOT -botcode2 NO_LEAVE_BOT -numgames 100
    autoplay stop

If a config file is provided, it is loaded first and any CLI flags override
individual fields. If no config file is given, all parameters come from flags
(defaulting to two HASTY_BOT players with default settings).

Experiments are defined in a JSON config file (protojson format).
When an experiment starts, three output files are created in the
configured output directory:

    {experimentId}.txt          -- per-turn log (rack, play, score, equity, ...)
    games-{experimentId}.txt    -- per-game summary (scores, bingos, who went first)
    {experimentId}.config.json  -- copy of the config for reproducibility

If no experiment_id is set in the config, one is auto-generated from the
current timestamp plus 6 random hex characters.

To stop a running autoplay: autoplay stop

Example config file (infer-tau001.json):

    {
      "description": "Test inference at tau=0.01, 5 plies, no endgame",
      "experimentId": "infer-tau001-5ply",
      "lexicon": "NWL23",
      "letterDistribution": "English",
      "numGames": 1000,
      "threads": 32,
      "outputDir": "./experiments",
      "player1": {
        "botCode": "SIMMING_INFER_BOT_NO_EG",
        "minSimPlies": 5,
        "simThreads": 6,
        "inferenceTau": 0.01
      },
      "player2": {
        "botCode": "SIMMING_BOT_NO_EG",
        "minSimPlies": 5,
        "simThreads": 6
      }
    }

Config fields:

    description        Human-readable label for the experiment (logged at start).
    experimentId       ID used for output file names. Auto-generated if empty.
    lexicon            Lexicon name (e.g. "NWL23", "CSW21"). Defaults to the
                       configured default lexicon.
    letterDistribution Letter distribution (e.g. "English"). Defaults to the
                       configured default.
    numGames           Number of games to play. Defaults to effectively unlimited.
    threads            Number of parallel game threads. Defaults to NumCPU.
                       Set to 1 if either bot uses endgame or pre-endgame solving.
    outputDir          Directory for output files. Defaults to current directory.
    block              If true, wait for all games to complete before returning.

    seedFile           Path to a seed file for deterministic/reproducible games.
    generateSeeds      If true, generate seeds and save to seedFile, then exit.
    deterministic      If true, load seeds from seedFile and use them for games.

Per-player fields (player1 / player2):

    botCode            Bot strategy. Values from the BotRequest.BotCode enum:
                         HASTY_BOT, NO_LEAVE_BOT,
                         SIMMING_BOT, SIMMING_BOT_NO_EG,
                         SIMMING_INFER_BOT, SIMMING_INFER_BOT_NO_EG,
                         HASTY_PLUS_ENDGAME_BOT,
                         LEVEL1_COMMON_WORD_BOT ... LEVEL4_COMMON_WORD_BOT,
                         LEVEL1_PROBABILISTIC ... LEVEL5_PROBABILISTIC
    leaveFile          Custom leave file (.klv2) inside data/strategy/<lexicon>/.
    pegFile            Pre-endgame adjustment file (.json) in the same dir.
    minSimPlies        Minimum Monte Carlo sim plies (default 2 for simming bots).
    simThreads         Threads for the bot's internal sim (default: all CPUs).
    stochasticStaticEval  If true, uses stochastic equity evaluation.
    inferenceTau       Softmax temperature for rack inference (0 = use default 0.05).
                       Lower = assume opponent plays more optimally.
                       Higher = allow more weight to sub-optimal plays.
                       Try 0.008–0.03 for strong opponents.

CLI overrides:

Any config field can be overridden on the command line without editing the
JSON file. Override flags are applied on top of the loaded config:

    autoplay base.json -numgames 500
    autoplay base.json -tau1 0.008
    autoplay base.json -threads 16 -experimentid infer-tau008-run2

Available override flags:
    -experimentid ID      Override the experiment ID (and thus output filenames)
    -outputdir PATH       Override the output directory
    -lexicon NAME         Override the lexicon
    -letterdistribution X Override the letter distribution
    -numgames N           Override the number of games
    -threads N            Override the number of parallel game threads
    -block                Wait for all games to complete
    -seedfile PATH        Override the seed file path
    -genseeds             Generate seeds and save to seedFile, then exit
    -deterministic        Load seeds from seedFile for deterministic games
    -botcode1 CODE        Override player1 bot code
    -botcode2 CODE        Override player2 bot code
    -minsimplies1 N       Override player1 minimum sim plies
    -minsimplies2 N       Override player2 minimum sim plies
    -simthreads1 N        Override player1 sim threads
    -simthreads2 N        Override player2 sim threads
    -tau1 F               Override player1 inference tau
    -tau2 F               Override player2 inference tau

Deterministic workflow example:

    # Generate seeds
    autoplay base.json -numgames 1000 -genseeds -seedfile /tmp/seeds.txt

    # Run on master branch with seeds
    autoplay base.json -numgames 1000 -deterministic -seedfile /tmp/seeds.txt \
        -experimentid master-run1

    # Run on feature branch with same seeds
    autoplay base.json -numgames 1000 -deterministic -seedfile /tmp/seeds.txt \
        -experimentid feature-run1

Note: If using a simming or inference bot, set threads to 1 and let
simThreads use all CPUs, or balance as needed (e.g. threads=32, simThreads=6
on a 192-core machine).
